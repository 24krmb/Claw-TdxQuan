# 量化分析与交易

一个简单的投资想法 => 可执行的交易策略 => 可执行的代码程序 => 检验交易策略效果 => 实盘交易验证改进

## 基础执行步骤

Step 1：从一个简单的投资想法开始

将用户认为可能会盈利的投资方法、理念，比如熊市时期银行股是潜力股、复制基金经理的增强指数、金叉买入死叉卖出等等。这些想法通常以网络、书本和讲座等为载体，来源于投顾、同行以及自己的经验总结等等。

Step 2：完善这个想法，形成明确的可执行的交易策略

如果用户的投资想法比较模糊，**必须跟用户询问清楚以下几点**：

**Security**：确定投资品种或范围（市场、板块、行业、风格）

**Condition**：确定触发买/卖的具体条件：

```mindmap
root((部分因子全景))
  基本面因子
    估值因子
    成长因子
    质量因子
  技术面因子
    动量因子
    波动率因子
    流动性因子
  另类因子
    情绪与舆情因子
      社交媒体情绪
      新闻情感分析
      分析师预期
    资金流与微观结构因子
      主力资金流向
      订单簿失衡
      高频交易模式
    事件驱动与主题因子
      特定事件冲击
      行业主题轮动
    宏观与另类数据因子
      宏观经济指标
      专利与招聘数据
      供应链与电商数据
```

**Quantity**：确定买卖的数量/金额等

明确的可执行的交易策略的判断基准：根据交易策略的描述，不同的人在相同情形下，都能做出相同的交易操作。

Step 3：编写代码，把交易策略转成可执行的代码程序

为了验证这个策略是否赚钱，把明确后的交易策略通过编程转成程序，根据**实时数据**执行该策略产生实盘交易。

Step 4：回测或者模拟交易，检验策略效果

通过回测和模拟交易两种方法检验策略效果

**回测举例说明：**

- **策略环境**：设定初始虚拟资产100万元；选择一段历史时间区间（20100101 到 20200101）；把该时间区间的各种数据（如收盘股价行情等）发给计算机。
- **策略执行**：计算机利用这些数据模仿历史真实的市场，执行我们编写的策略程序。
- **策略评估**：计算机会出具一份报告，根据这个报告我们知道，在 20100101 期初的 100 万元，按照我们的策略交易到期末 20200101，会怎样？一般包括盈亏情况、下单情况、持仓变化以及一些统计指标等，根据此评估交易策略的好坏。
- **回测案例**：[skills/TQ-Python/scripts/VBT.py](skills/TQ-Python/scripts/VBT.py)（VBT 简单回测并输出图形）

**模拟交易举例说明：**

- **策略环境**：设定初始的虚拟资产比如100万元，选择开始执行模拟交易的时间点，比如下周一。那么从下周一开始，股市开始交易，真实的行情数据就会实时地发送到计算机。
- **策略执行**：计算机利用真实的数据模仿真实的市场，执行你的策略代码输出买卖队列，模拟系统会记录每一笔买卖记录。
- **策略评估**：我们可以得到一份实时更新的策略评估报告，这报告类似于回测得到的报告，不同的是会根据实际行情变化更新；同样我们能据此评估交易策略的好坏。

Step 5：实盘执行交易策略，并持续优化改进策略

⚠注意，这时不再是用虚拟资产进行模拟交易，实盘交易账户上的盈亏都是真金白银。

实盘交易一般也会给出一份类似模拟交易的投资分析报告，通过实时观察策略的实盘表现、根据投资理念的变化、市场状况的变化及时修正、改善和优化策略，使之保持持续盈利能力。
